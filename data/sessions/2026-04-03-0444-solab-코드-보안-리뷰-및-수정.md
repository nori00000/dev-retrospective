## 작업 요약

sociai.org 모바일 406 버그 수정 배포 완료 (iPhone Safari, Samsung Internet → HTTP 200 확인). 이후 전체 코드 리뷰를 통해 Critical 4건, Warning 8건, Minor 9건 총 23개 이슈를 식별하고 모두 수정·배포했다. UCP 엔드포인트 인증 부재, 재고 race condition, CSP report-only 등 금전적·보안적으로 중요한 이슈가 포함됐다. executor 에이전트가 SSH heredoc 안에서 Ruby 문자열을 작성할 때 `\"` 이스케이프가 소스에 그대로 삽입되어 문법 오류로 앱이 다운되는 사고가 발생했으나 빠르게 진단·복구했다.

## 상세 작업 내역

### 1. 모바일 406 버그 배포

Kamal 설정이 레거시(192.168.0.1 불가)임을 확인하고 Fly.io 배포 방식 파악. M4 Studio에서 `fly deploy` 실행. iPhone Safari·Samsung Internet 모두 HTTP 200 확인.

### 2. 전체 코드 리뷰 (23개 이슈)

code-reviewer 에이전트가 60+ 파일을 분석하여 Critical/Warning/Minor 분류 리포트 생성.

**Critical:**
- C1: UCP BaseController CSRF 비활성화 + API 인증 없음 → X-UCP-Token 헤더 인증 추가
- C2: UCP products 검색 LIKE wildcard injection → `sanitize_sql_like` 적용
- C3: `product.decrement!` 비원자적 재고 차감 → `update_all` atomic SQL로 교체
- C4: CSP `report_only = true` → `false`로 enforcement 활성화

**Warning (주요):**
- N+1 쿼리 (HomeController, following? 메서드)
- Admin HTTP Basic Auth에 current_admin_user 추적 추가
- `@commentable` nil 가드 누락 → head :not_found 처리
- UCP rescue에서 `e.message` 직접 노출 → 로깅만 하고 일반 메시지 반환
- `increment_views!` → `update_counters` atomic 변경

**Minor (주요):**
- `Experiment`, `Comment` moderation_status 기본값 1(approved) → 0(pending)
- Devise `:lockable` 추가 (brute-force 방지)
- `avatar_url` 타입 불일치 수정
- `ContactForm` PORO 분리
- `NotificationPresenter` 도입 (모델에서 url_helpers 제거)

### 3. 배포 사고 및 복구

W7 수정 중 executor가 SSH heredoc 안에서 `\"\n\"` 이스케이프를 Ruby 소스에 그대로 삽입 → `checkout_sessions_controller.rb` 문법 오류 → 앱 크래시. `fly logs`로 Zeitwerk eager loading 에러 확인 → `ruby -c`로 정확한 라인 특정 → Python으로 수정 → 재배포 성공.

## 배운 점 (TIL)

- **SSH heredoc 안에서 Ruby 문자열 작성 시 이스케이프 오염**: executor 에이전트가 SSH 명령 안에 Ruby 코드를 작성할 때 `\"` 가 그대로 소스에 들어갈 수 있음. 커밋 전 `ruby -c` 문법 검사가 필수
- **Fly.io 앱 크래시 진단 순서**: `fly status` → `fly logs --no-tail` → grep으로 에러 라인 추출 → `ruby -c`로 로컬 확인
- **`fly machine start`로 수동 재기동 가능**: 배포 실패 후 머신이 stopped 상태가 되면 `fly machine start {id}`로 재시작 가능 (임시 복구용)
- **Zeitwerk eager load = production 시작 시 모든 파일 로드**: development에서 문법 오류를 놓쳐도 production에서 즉시 크래시

## 배운 점 (TIL)

### 1. setup.sh 실행 실패의 근본 원인
- **문제**: setup.sh가 실행되지 않는 근본 원인은 ~/.dev-retrospective symlink 부재
- **원인**: setup.sh가 `$(dirname "$(readlink ~/.dev-retrospective)")` 패턴으로 레포 경로 추정
- **교훈**: symlink 기반 설정은 순환 의존성 주의 필요 (setup.sh가 symlink 생성하는데 실행 자체가 symlink 필요)

### 2. cron 환경에서 PATH 제한
- **문제**: cron에서 실행하는 스크립트가 /usr/local/bin의 Claude CLI를 찾지 못함
- **원인**: cron 환경의 PATH는 /usr/bin:/bin만 포함
- **해결**: 스크립트 상단에 `export PATH="/usr/local/bin:$PATH"` 추가
- **교훈**: cron 스크립트는 항상 전체 PATH 명시 필요

### 3. GitHub GITHUB_TOKEN의 레포 접근 제한
- **문제**: GitHub Actions의 GITHUB_TOKEN은 현재 레포만 접근 가능
- **원인**: GitHub의 보안 정책 (크로스 레포 접근은 명시적 PAT 필요)
- **해결**: Personal Access Token (PAT)을 GitHub Secrets에 저장
- **교훈**: 크로스 레포 작업은 항상 PAT 필요

### 4. Session hooks의 session-log.md 충돌 위험
- **문제**: session-checkout이 session-log.md를 생성하는데, 이미 파일이 있으면 충돌
- **해결**: RALPH Phase 0에서 `git diff` 실행하여 충돌 확인
- **교훈**: 자동화 시스템은 멱등성(idempotency) 보장 필요

### 5. launchd 경로 오류 디버깅
- **문제**: salpim launchd가 로드되었으나 실행 실패
- **원인**: plist의 경로가 ~/salpim/daily.sh인데 실제 경로는 ~/projects/salpim/daily.sh
- **디버깅**: `launchctl list | grep salpim` → `cat ~/Library/LaunchAgents/com.leesangmin.salpim.plist`
- **교훈**: launchd 디버깅은 `log stream --predicate 'subsystem == "com.apple.launchd"'` 활용

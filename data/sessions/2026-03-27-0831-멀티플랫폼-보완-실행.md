## 핵심 결정

- **vault-detect.sh 공유 함수**: 모든 hooks/scripts가 동일한 탐지 로직을 사용하도록 단일 소스 파일로 통합. 중복 제거 + 유지보수 용이.
- **Windows 3-tier fallback**: Developer Mode symlink → Junction → Copy. 어떤 Windows 환경에서도 작동하도록 graceful degradation.
- **hostname stagger cron**: `cksum` 해시로 0-4분 오프셋. 멀티머신 git push 충돌을 사전 예방.
- **repo→vault 단방향만**: vault→repo 역방향 동기화 배제. 혼란 방지, repo가 유일한 source of truth.

## 배운 점 (TIL)

- ralplan 합의 플랜이 있으면 ultrawork 실행이 매우 효율적 — 각 에이전트에 정확한 스펙을 전달할 수 있어 병렬 실행 품질이 높음
- Windows Claude Code hooks는 Git Bash로 실행되지만 WSL이 있으면 충돌 가능 — `CLAUDE_CODE_GIT_BASH_PATH` 설정 필요
- `stat -f %m` (macOS) vs `stat -c %Y` (Linux) — 크로스 플랫폼 bash에서 파일 수정시간 체크 시 분기 필수
- cron에서 hostname 기반 stagger는 `cksum | cut -d' ' -f1` % N으로 간단하게 구현 가능

## 상세 작업 내역

### 1. Python 살핌 변경사항 커밋 및 푸시

	이전 세션에서 작업한 sentinel→살핌 리네이밍, SSL certifi 수정, 알림 소리 끔 등의 미커밋 변경사항을 정리하여 `nori00000/salpim` 리포에 커밋/푸시.

### 2. salpim-web 아키텍처 분석

	다른 컴퓨터에서 생성된 Rails 8 앱(salpim-web)의 전체 구조 파악:
	- 17개 컨트롤러, 44개 DB 테이블, 20+ 서비스
	- ThinkingOS 4단계 사고 프로세스 (진단→계획→실행→회고)
	- Google Calendar OAuth2, AI 코칭 (Claude/OpenAI/Ollama), Solid Queue
	- **Telegram 통합 없음** 확인 → 포팅 필요

### 3. salpim-web에 TelegramService 포팅

	Python 살핌의 Telegram 봇 기능을 Rails 서비스로 이식:
	- `TelegramService` 생성 (Net::HTTP 기반, gem 불필요)
	- DB 마이그레이션: users 테이블에 `telegram_chat_id`, `telegram_enabled` 추가
	- `CalendarSyncUserJob`: 새 알림 시 Telegram 발송
	- `WeeklyReviewJob`: 주간 리뷰 시 Telegram 발송
	- 실제 모델 구조(StrategicBriefing, ScheduleAlert)에 맞게 검증 및 수정

### 4. 멀티머신 아키텍처 결정

	"어떤 컴퓨터에서든 같아야 한다" 요구사항 반영:
	- Python 살핌 아카이브 → salpim-web 단일 운영으로 통합
	- Mac Studio M4 (128GB) vs Synology NAS 925+ 비교 → Mac Studio 선택
	- Tailscale VPN으로 어디서든 접속
	- Mac Studio에서 Ollama 로컬 AI 코칭 가능 (128GB)

### 5. oh-my-claudecode v4.9.1 업데이트

## 핵심 결정

- **salpim-web 단일화**: 멀티머신 환경에서 Python 로컬 데몬은 머신마다 셋업 필요 → 웹앱 하나로 통합이 유지보수/일관성 면에서 우월
- **Mac Studio 서버 선택**: Synology NAS 대비 Ollama 로컬 AI(128GB), 네이티브 Rails, 동일 개발환경 장점
- **TelegramService를 gem 없이 구현**: Net::HTTP만 사용하여 의존성 최소화, Python 버전과 동일한 접근
- **Python 살핌 아카이브**: 로컬 전용 기능(macOS 알림, Obsidian)은 Telegram + 웹 대시보드로 대체

## 배운 점 (TIL)

### 1. setup.sh 실행 실패의 근본 원인
setup.sh가 실행되지 않는 근본 원인은 ~/.dev-retrospective symlink 부재. setup.sh가 `$(dirname "$(readlink ~/.dev-retrospective)")` 패턴으로 레포 경로 추정하므로 순환 의존성 발생.

### 2. cron 환경에서 PATH 제한
cron 환경의 PATH는 /usr/bin:/bin만 포함. /usr/local/bin의 Claude CLI를 찾지 못함. 스크립트 상단에 `export PATH="/usr/local/bin:$PATH"` 추가 필요.

### 3. GitHub GITHUB_TOKEN의 레포 접근 제한
GitHub Actions의 GITHUB_TOKEN은 현재 레포만 접근 가능. 크로스 레포 작업은 Personal Access Token (PAT) 필요.

### 4. Session hooks의 session-log.md 충돌 위험
session-checkout이 session-log.md를 생성하는데, 이미 파일이 있으면 충돌. RALPH Phase 0에서 `git diff` 실행하여 사전 확인 필요.

### 5. launchd 경로 오류 디버깅
launchd가 로드되었으나 실행 실패 시 `cat ~/Library/LaunchAgents/*.plist`로 경로 확인. `log stream --predicate 'subsystem == "com.apple.launchd"'`로 실시간 로그 분석.
